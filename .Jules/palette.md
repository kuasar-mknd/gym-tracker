## 2026-03-25 - Accessibility for Icon-only Buttons
**Learning:** Icon-only buttons (like delete 'X' or trashcan icons) within dynamic lists or tool pages often lack semantic context for screen readers. Using `aria-label` alongside `title` provides both screen reader accessibility and helpful hover tooltips for sighted users.
**Action:** Always check interactive elements that rely solely on icons (like Material Symbols) and ensure they have a descriptive `aria-label` and ideally a `type="button"` attribute to prevent accidental form submissions.
## 2026-03-26 - Keyboard Accessibility for Clickable Divs\n**Learning:** Interactive elements implemented as `<div>` with `@click` handlers are not accessible to keyboard users by default. To make them accessible without changing the DOM structure (e.g., to a `<button>`), they require `role="button"`, `tabindex="0"`, and explicit keyboard event listeners like `@keydown.enter` and `@keydown.space.prevent`.\n**Action:** Always check interactive `<div>` or `<span>` elements used for actions (like selecting items from a list or expanding cards) and ensure they include proper ARIA roles, tabindex, and keyboard event handlers. Consider adding `outline-none focus-visible:ring-2` for visual focus feedback.
